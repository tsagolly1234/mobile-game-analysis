# 🎮 Анализ мобильной игры

## О проекте
Тестовое задание компании, разрабатывающей мобильные игры.
Менеджер поставил три независимых задачи по исследованию ключевых аспектов приложения.

## Задачи

### 1. Retention игроков
Написала на Python функцию `retention_n_day()`, которая считает ежедневный retention
по дням с даты регистрации игрока и визуализирует результат в виде тепловой карты когорт.

### 2. A/B тест акционных предложений
Проанализировала результаты A/B теста двух наборов акционных предложений.  
Несмотря на то что ARPU в тестовой группе выше на 5%, выводы неоднозначны:
- Конверсия в оплату в тесте упала на 6% статистически значимо (тест Хи-квадрат)
- В тестовой группе полностью отсутствует сегмент крупных покупателей
- Различия в ARPU и ARPPU статистически незначимы (t-тест, p ≈ 0.5)

**Вывод:** тестовый набор предложений не рекомендован к раскатке.

### 3. Метрики игровых событий (Plants & Gardens)
Предложила систему метрик для оценки тематических событий: воронка прохождения уровней,
DAU, retention участников, конверсия в оплату, баланс ресурсов.  
Адаптировала набор метрик под усложнённую механику с откатом прогресса
добавила churn rate, долю успешных попыток, долю откатов и мониторинг обращений в поддержку.
